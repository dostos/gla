This SHA is the *pre-fix* tree for three.js issue #33060 ("Feedback loop
error with transmission + antialias:false + DoubleSide since r182").
The fix landed in PR #33063.

We pin the pre-fix SHA so the eval scenario reproduces the buggy
configuration: MeshPhysicalMaterial({transmission:1, side:DoubleSide})
on a WebGLRenderer({antialias:false}). On any backend lacking the
WEBGL_multisampled_render_to_texture extension, the back-side
transmission pass produces a feedback loop because the transmission
sampler texture is the bound framebuffer's COLOR_ATTACHMENT0.
