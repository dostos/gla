# Round R13 (2026-05-05)

First round under the per-round-log convention with explicit lift
attribution. Validates two R12c backlog items (P0 + P1) that landed
just before launch.

## Ran

- Cohort: same 14 R12 scenarios as R12c/R12d (8 godot + 6 web-map)
- Modes: `with_gla`, `code_only`
- Model: `claude-opus-4-7[1m]` via claude-cli
- Output: `/data3/gla-eval-results/2026-05-05-r13-scope-hint/`
- Variable from R12c baseline: P1 scope hint shipped (`1007d10`),
  P0 mining-quality verifier shipped (`ad0af3e`); both available in
  the running harness via `git tag round-r13-start = ad0af3e`.

## Findings

- **The scope hint works.** `file_solved` went from 0/14 (R12c, both
  modes) to 4/14 with_gla and 6/14 code_only. Agents now name enough
  ground-truth files to clear the 0.5 recall threshold *without*
  needing the LLM-judge fallback. Direct empirical evidence that
  calibrating the agent's search scope prevents the file-mismatch
  failure mode identified in R12c forensics.
- **Token spend dropped 25–48%.** with_gla: 147k → 111k output tokens
  (-25%); code_only: 162k → 84k (-48%). The hint reduces grinding —
  agents stop reading 30 files when the canonical fix is 3.
- **Net solve rate up, with one stochastic regression.** Combined
  20/28 → 21/28 (+1). `godot_performance_on_android` flipped to
  solved in BOTH modes (was unsolved both in R12c) — the scope hint
  helped the agent find the actual Vulkan/Mali code path despite
  the (still-incorrect) Android-Java-heavy fix.files. One reverse
  flip: `godot_weird_shadow` was wg-solved/co-failed in R12c, now
  wg-failed/co-solved — consistent with the stochastic-variance
  pattern documented in R12c (claude-cli grepped different keywords).
- **with_gla < code_only in R13** (9 vs 12). The gap widened. With
  the scope hint, code_only agents converged faster than with_gla
  agents — the GPA tool block adds prompt overhead that the scope
  hint partly compensates for in code_only but not in with_gla.
  Hypothesis: `with_gla` agents still spend turns enumerating GPA
  endpoints that don't help (browser-tier scenarios trigger the
  warning but the warning doesn't gate the GPA tool block in the
  prompt). Worth a P3-tier follow-up: when browser-tier warning
  fires, drop the GPA tool block from the prompt entirely.
- **godot_4_2_world_environment** still fails in both modes
  (file_level/judge=none). The scope hint correctly told the agent
  "13 files under servers/rendering/renderer_rd/", but the agent
  *still* converged on the luminance_multiplier story instead of the
  buffer-format change. Reasoning-shallow remains the dominant
  remaining failure mode for this scenario.

## Side observations

- The scope hint moved scenarios from `judge`-rescue to `file_level`
  direct solves. That's the cheap-scoring path: when agents name
  enough files, no LLM-judge call is needed. Effective cost reduction
  on top of the token savings.
- Browser-tier warnings fired correctly for all 6 web-map scenarios
  in with_gla mode. The warning surfaces in `with_gla.log`; consider
  promoting it to a structured field on EvalResult so it's queryable
  in the report.
