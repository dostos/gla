# Round R12d (2026-05-05)

A/B test of strengthened JSON output prompts. Same 14 scenarios, same
agent model, but with the prompt rewritten to lead with a "READ FIRST
— non-negotiable" output contract. Result: regression. Reverted.

## Ran

- Cohort: same 14 R12 scenarios as R12c
- Modes: `with_gla`, `code_only`
- Model: `claude-opus-4-7[1m]` via claude-cli
- Output: `/data3/gla-eval-results/2026-05-05-iter-r12d-json/`
- Variable: prompts in `src/python/gpa/eval/prompts/` rewritten with
  the JSON output contract leading the file (vs trailing in R12c).

## Findings

- **The heavy "READ FIRST" prompt collapsed agent investigation 5×.**
  Average output tokens: R12c 10,526 → R12d 2,279. Tool calls: R12c
  ~25 → R12d ~10. Agents prioritized emitting JSON over investigating.
- **Solve rate dropped 10/14 → 2/14 (with_gla) and 10/14 → 4/14
  (code_only).** Short responses can't satisfy the prose-recall
  threshold for multi-file fixes (godot fixes touch 13 files; agent
  must name 7+ to clear 0.5 recall).
- **Parsed-JSON rate did improve** (0/14 → 4/14 with_gla, 8/14
  code_only). The prompt change *did* push agents toward JSON. Just
  at the cost of investigation depth.
- **Agents emit JSON when asked clearly enough — but if you make it
  the first/loudest instruction, they treat it as the primary task.**
  The lighter R12c framing (output contract at the end as a footnote)
  is the right balance.
