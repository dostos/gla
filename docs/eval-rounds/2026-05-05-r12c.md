# Round R12c (2026-05-05)

First "real" round after the eval infrastructure rewrite. Same 14
scenarios as R12b, same model, but with the snapshot pipeline + scoring
stack actually working.

## Ran

- Cohort: 14 scenarios (same set as R12b — 8 godot + 6 web-map,
  curated from `r5211bd_*` and `rfc2ac5_*` mining batches)
- Modes: `with_gla`, `code_only`
- Model: `claude-opus-4-7[1m]` via claude-cli
- Output: `/data3/gla-eval-results/2026-05-05-iter-r12c-rerun/`

## Findings

- **R12b numbers were artifactual.** The snapshot pipeline silently
  served the *post-fix* tree (because `fix_parent_sha` defaulted to
  `fix_sha` when missing), so agents investigated already-fixed code.
  Real R12c numbers: 10/14 in both modes, vs R12b's reported 1/14.
- **Three silent scoring bugs** masked correct diagnoses for the
  entire R12 series:
  1. Prose-scorer regex `c|cc|cpp` matched `c` first → every `.cpp`
     path truncated to `.c` and missed ground truth.
  2. LLM-judge gate was `scorer == "prose" AND needs_review` only —
     a correct diagnosis citing 3/13 ground-truth files (file score
     0.23) got dropped to `no_signal` and never reached the judge.
  3. `git show <fix_sha>` against a depth-1 snapshot couldn't see
     the merge's parent → reported "13506 files / 7M insertions"
     instead of the actual 13-file fix → judge couldn't compare.
- **with_gla = code_only at 71% solved** (10/14 each), but with_gla
  used 90% of the tokens. Native godot scenarios benefit; web-map
  ties (browser WebGL not intercepted by the native shim).
- **Token-spend predicts confidence**: solved scenarios use 8k output
  tokens on average; failed ones use 16k. Agents grind when stuck.
- **Browser-tier mismatch is real**: 2 maplibre WebGL scenarios where
  the GL shim sees nothing — agent enumerates empty overviews and
  gets distracted. Tier-mismatch warning landed in `d7bd4bb`.
- **Forensic taxonomy of 5 distinct failures** (3 fail in both modes
  + 2 mixed):
  - 2/5 scenario quality: mining picked one fix when multiple PRs
    closed the same issue (`cesium_camera_jumps`,
    `godot_performance_on_android`)
  - 1.5/5 file-mismatch with correct reasoning
    (`godot_4_2_world_environment`, `godot_wrong_position`)
  - 1/5 reasoning-shallow (`godot_4_2_world_environment` partial)
  - 1/5 stochastic agent variance (`godot_weird_shadow`)
